# Interview Coder

All-in-one SWE interview prep platform: DSA, LLD, HLD, Behavioral -- with code editor, diagrams, AI hints, spaced repetition, and a curated learning library.

## Tech Stack

- **Framework**: React 19 SPA with React Router v7
- **Build**: Vite 7 (ESM, `type: "module"`)
- **Language**: TypeScript (strict: false)
- **Styling**: Tailwind CSS v4 (via `@tailwindcss/vite` plugin)
- **Database**: TursoDB (libSQL) -- migrated from Supabase
- **Auth**: Google One Tap (OAuth2) + JWT tokens -- no redirect flow
- **Backend**: Vercel serverless functions (`api/` directory, `.mjs` files)
- **Local dev server**: Express on port 3456 (`server/index.mjs`) -- CLI bridge for claude/codex/gemini
- **Code editor**: Monaco Editor (`@monaco-editor/react`)
- **Diagrams**: Excalidraw
- **AI**: Multi-provider streaming (OpenAI, Anthropic, Google, DeepSeek, Qwen, + local CLI tools)
- **Analytics/Feedback**: SASS Maker SDK
- **Deployment**: Vercel

## Architecture

```
src/
  main.tsx              # Entry: BrowserRouter > AuthProvider > App
  App.tsx               # Route definitions, category routing
  types.ts              # All shared types (Problem, InterviewCategory, etc.)
  contexts/
    AuthContext.tsx      # Google OAuth + guest mode, JWT token management
    CategoryContext.tsx  # URL-param-driven category (dsa/lld/hld/behavioral)
  hooks/
    useAI.ts            # Multi-provider AI streaming, chat persistence
    useCodeExecution.ts # Sandboxed JS/TS execution via iframe + sucrase transpiler
    useSpacedRepetition.ts # SM-2 algorithm, localStorage-backed
    useProgress.ts      # Problem status/bookmarks/code saving, localStorage
    useProblems.ts      # Static JSON + user-imported problems, lazy category loading
    useNotes.ts         # Per-problem notes, backend or localStorage
    useLibrary.ts       # External repo content loader with quality scoring
  pages/
    Home.tsx            # Landing/category selection
    Dashboard.tsx       # Problem list + progress stats per category
    ProblemView.tsx     # Main workspace: editor + diagram + AI chat + notes
    Patterns.tsx        # Pattern-grouped problem browser
    AnkiReview.tsx      # Spaced repetition flashcard review
    ImportProblem.tsx   # LeetCode problem importer
    Library.tsx         # Curated learning repos browser
    RepoView.tsx        # Single repo reader with section tree + exercises
    Playground.tsx      # Freeform code playground
    VibeLearning.tsx    # AI-guided learning mode
  components/           # Layout, DiagramEditor, MarkdownViewer, etc.
  adapters/             # Library adapters (markdown-to-section-tree parser)
  lib/
    leetcode.ts         # LeetCode API integration
    diagramToText.ts    # Excalidraw diagram to text description
  data/
    problems.json       # DSA problems (bundled)
    lld-problems.json   # LLD problems (lazy-loaded)
    hld-problems.json   # HLD problems (lazy-loaded)
    behavioral-problems.json
    library/            # Generated by scripts/fetch-library.mjs at build time

api/                    # Vercel serverless functions
  auth/google.mjs       # POST: verify Google credential, create/find user, return JWT
  auth/verify.mjs       # GET: verify JWT
  chat.mjs              # POST: CLI bridge (spawns claude/codex/gemini, streams SSE)
  chats.mjs             # GET/POST/DELETE: per-problem chat history (TursoDB)
  notes.mjs             # GET/POST: per-problem notes (TursoDB)
  problems.mjs          # GET/POST: user-imported problems (TursoDB)
  db/client.mjs         # TursoDB client singleton
  db/schema.mjs         # Auto-create tables on first request

server/                 # Local dev Express server (CLI bridge only)
  index.mjs             # POST /chat -> spawn CLI tools, SSE streaming
```

## Key Conventions

- **Hooks pattern**: All stateful logic in custom hooks -- components are thin wrappers
- **API endpoints**: Vercel convention -- default export `handler(req, res)`, `.mjs` extension
- **Auth guard**: `requireAuth(req, res)` returns user object or sends 401
- **Dual storage**: Every data hook has authenticated (API) and guest (localStorage) code paths
- **Streaming**: All AI uses SSE (`text/event-stream`) with `data: {json}\n\n` format
- **No Redux/Zustand**: React state + localStorage

## Commands

```bash
npm run dev              # Express CLI bridge (:3456) + Vite (:5173) concurrently
npm run dev:frontend     # Vite only
npm run server           # Express CLI bridge only
npm run build            # vite build -> dist/
npm run fetch-library    # Clone repos, parse markdown, write JSON
npm run lint             # ESLint
```

Vercel build: `npm run fetch-library && npm run build`

## Environment Variables

```bash
GOOGLE_CLIENT_ID=           # Backend Google OAuth client ID
VITE_GOOGLE_CLIENT_ID=      # Frontend (same value, Vite-exposed)
TURSO_DATABASE_URL=         # libsql://your-db.turso.io
TURSO_AUTH_TOKEN=           # TursoDB auth token
JWT_SECRET=                 # Random string for signing tokens
VITE_SAASMAKER_API_KEY=     # SASS Maker analytics/feedback
```

Guest mode works with zero env vars -- all data stays in localStorage.
